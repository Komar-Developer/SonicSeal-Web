SonicSeal — Privacy Policy

Effective date: 26 May 2026 · Last updated: 26 May 2026

TL;DR

SonicSeal is a 100% offline desktop application. We do not collect, transmit, or store any user data. No analytics, no telemetry, no crash reporting, no third-party tracking.

Data we do NOT collect

  • Personal information (name, email, address, phone, IP, device identifiers)
  • Audio files you process — they never leave your Mac
  • License keys — stored locally in macOS Keychain only
  • Usage analytics (clicks, time-on-app, feature usage)
  • Crash reports — logs are written only to ~/Library/Logs/SonicSeal and are never uploaded
  • Cookies or browser fingerprints

How the app works

SonicSeal runs a local FastAPI sidecar process bound exclusively to 127.0.0.1 (loopback). The Electron UI talks to this sidecar over the loopback interface only. No external network connection is made by the app for its core functionality.

Two optional network connections may occur, both initiated by you:

  1. License activation (only if you enter a license key): contacts the licensing server to validate your key once. Re-validation runs locally from a signed JWT — no further network requests required.
  2. Auto-update check (Gumroad / Developer ID build only): the Electron updater checks GitHub Releases for a newer version of SonicSeal. This is a standard HTTPS request to api.github.com and contains no user data beyond the standard HTTP user agent. Disabled in the Mac App Store build — App Store updates are delivered by Apple instead.

Where your data lives

All data SonicSeal creates lives in:

  • ~/Library/Application Support/sonicseal-desktop/ — license, SQLite DB with your local licensing records, sealed audio blobs
  • ~/Music/SonicSeal/ — default output folder for forged audio files; configurable from File menu → Output Folder
  • macOS Keychain — encrypted license token

None of this data is ever uploaded, synced, or shared.

Children's privacy

SonicSeal is a professional tool for music producers and DJs and is not directed at children under 13. Because we collect no personal data, we cannot inadvertently collect data from minors.

Changes to this policy

We may update this policy if the architecture of the app changes. Any update will increase the "Last updated" date at the top of this document. Material changes will also be announced in the in-app release notes.
